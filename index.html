One chat. Five experiences. Strict guardrails.

Bina (Hebrew: understanding) is Souled's in-house AI assistant. Every Olami/Souled staff member gets a chat window tailored to exactly what their role should see — and nothing they shouldn't.

Coaches get a sidekick for handbook lookups, their caseload, and student history. Regional directors get scoped student data. Fundraisers get aggregate counts and ready-to-share lists. Exec gets full access. One codebase. Five signed-in realities.

What Bina can do

Nine distinct tool capabilities. Which ones are available depends on the signed-in persona.

💛 Coach-only tools
📚Search the coaching KB

Handbook · Adina Q&A · Rivkie Y.I.D · coach meetings · WhatsApp nuggets · Souled wiki — indexed and cited on every answer.

📋Look up my own caseload

Four views — schedule, overdue notes, hours last 90 days, pay summaries last 12 weeks.

👥List my students

Current roster with last meeting date and next scheduled meeting — drives disambiguation when a coach refers to a student by first name only.

🔍Query one of my students

Full bundle — contact, relationship history across coaches, touchpoint narratives, trips, classes. Roster check runs fresh every call.

🏘️Find local connections

Mekarvim and Olami-umbrella educators by city, state, country, or role. Returns real emails and phone numbers.

🗓️Look up upcoming events + classes

Live Salesforce calendar — Shabbatons, retreats, Souled Circles, coach staff meetings. Dates, location, Zoom links, flyers.

🚨Flag for admin review

If a coach reports mental-health distress, Bina responds supportively and immediately emails the admin team.

👑 Exec · 🗺️ Regional · 📊 Analyst · 📈 Aggregate tools
🧠Run a Salesforce query

Claude writes SOQL from a plain-English question. Guardrails check it before it runs — allowed objects, allowed fields per persona, aggregate-or-row, Test contacts auto-excluded.

🪄Escalate what she can't answer

Qualitative questions, undefined glossary terms, off-scope asks — she flags the conversation, notifies the Souled Data Team, and tells the user a human will follow up.

Automatic QA & safety

Five live alerts. Six event kinds logged on every turn. Two rounds of adversarial stress-testing. And a data-sanity detector running on every coach query.

Live alerts (email dispatch via Resend)

🚨
flag_admin · immediate

A coach reported student mental-health distress. Admin team is emailed within seconds — no debounce.

💸
spend_spike · 60-min debounce

Day-to-date cost is more than 2× the trailing-7-day average. Catches runaway agents or new abuse.

⚡
error_burst · 30-min debounce

≥ 5 Salesforce or tool-loop errors in 15 minutes. Fires separately per error kind.

👤
user_abuse · 30-min debounce

Single user over 100 turns/hour or $10/day. Separate alerts for volume and cost.

🔌
claude_api_failure · 30-min debounce

≥ 3 Claude API errors in 5 minutes. Catches upstream outages fast.

📊
Debouncing survives restarts

Every alert dispatch writes an alert_sent row so a container restart won't re-fire within the debounce window.

SOQL guardrails — four layers before any query hits Salesforce

1. Object allowlist
Only 10 objects allowed — Contact, Relationship__c, Registration__c, Touch_Point__c, … Exec gets 4 additional (Workday, Pay_Period, Coach_Times, Supervision).
2. Field blocklist per persona
Case-insensitive substring match against the SELECT list — blocks phone, names, pay, narratives, AI reasoning depending on the caller's persona.
3. Aggregate-or-GROUP-BY
Fundraiser-Aggregate must include COUNT/SUM/AVG or GROUP BY. Geography carve-out allows MailingCountry/State/City inside GROUP BY only.
4. Test-data exclusion + injection defense
Test_Old__c = false auto-injected on Contact queries. Multi-statement queries, DML tokens (UPDATE/DELETE/INSERT/MERGE) rejected.

Event logging — every turn, every tool call

tool_call
Every tool invocation logged with inputs and outcome.
cannot_answer
When Bina escalates — shows up in Quality tab.
flag_admin
Mental-health distress reports. Drives the immediate alert.
guardrail_reject
A SOQL query was rejected before running. Helps improve the agent's query-writing.
sf_error
Salesforce returned an error. Burst of these triggers the error_burst alert.
tool_loop_exhausted
Agent hit the 8-turn tool loop cap — typically means the question is ill-formed.

Data-sanity detector. A student is on a coach's roster if the relationship is active — but "active" has two independent definitions in Salesforce (end-date-based vs status-based). Bina computes both predicates on every query and logs any divergence to /api/admin/data-sanity. This means the system catches data-quality problems as a side effect of normal use.

Technical wins

Prompt caching with ephemeral scope

The large persona system prompt + glossary is marked cache_control: ephemeral. Every subsequent turn in the same conversation reads it for ~10% of the cost of writing it. Cache-hit rate shown in the Cost tab.

Roster gate — never cached

When a coach asks about a student, we re-verify the roster membership with fresh SOQL every single call. Prevents the LLM from passing a student ID from a prior turn that no longer applies.

Coach KB hot-swap

Admin uploads a new zip of handbook + transcripts via the admin panel. The index rebuilds in-process. No deploy, no downtime. Works great for urgent handbook updates.

Analytics isolated from chat

SQLite for telemetry runs in a separate database from conversations.json. If analytics fails, chat keeps working. BINA_ANALYTICS=off kills all writes instantly for rollback.

Cost math at insert time

Per-turn cost computed from the then-current model-prices table and stored. Future price changes don't retroactively distort historical dashboards.

Glossary-driven honesty

The glossary defines every entity and metric Bina can answer about. Ask about something undefined — "committed to marry Jewish" — and she says "that's not defined yet" instead of guessing.
